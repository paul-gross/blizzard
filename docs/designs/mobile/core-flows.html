Flow decisions

The push notification deep-links to the ask itself, never the board — speed-to-done is the whole game. Options render as full-width 48px tap targets with the agent's suggested answers first; freeform is the fallback, not the default.

Context is a fixed four-row card (chunk · node · runner · spend), not the full detail panel — enough to answer confidently, one tap away from more.

The confirmation screen immediately surfaces the next waiting ask: clear the queue in one session, then pocket the phone.

Reuses questions.query, QuestionView options, compactRef, answer POST from chunk-awaiting-human. New: mobile ask shell, tab bar.

Flow decisions

Ordered by attention, not by entity: needs you first (open asks + needs_human chunks folded into one list), then what's in motion, then the comfort numbers. The vitals strip answers "am I needed?" before a single scroll.

Zero taps required for the answer to be no — the whole point of the restaurant check. Every row is still a tap-through to the ask flow or the run story.

The live dot is the SSE spine's health; on visibilitychange the stream reconnects and catches up before re-rendering, so a backgrounded tab can never show stale rows as fresh.

Reuses chunks.query (ChunkSummary), questions.query, health, fleet-spend, cost-format, status vocabulary from chunk-lanes. New: attention-sort, vitals strip, mobile board shell.

Flow decisions

Read-optimized, because this is the demo surface: the timeline is the hero, telling the run as a narrative — each hop carries its verdict and its lap cost, so "the reviewer bounced it once and it cost $0.83" is legible to a stranger holding your phone.

Transcripts page turn-by-turn (pager + swipe) instead of one infinite scroll — a show-off wants "look at this turn", not a wall. UTC clock stamps kept verbatim from the local-panel's rendering rule.

Scoping note: transcript reads are the runner's API (lease-keyed, via local-panel's query today). The mobile hub shell needs a proxied route to the owning runner — the one genuinely new plumbing item in these mocks.

Reuses chunk-timeline row model + verdict colors, chunk-facts, transcript.query, formatUtcClock, cost-format. New: mobile run-story shell, turn pager.

Shared-guts inventory across all five screens: design tokens verbatim (no new colors, no new type steps), compactRef, the status pill vocabulary, every query/service, the SSE spine. The net-new surface is four mobile shells (ask, board-glance, run-story, transcript), a ViewportService + manual override, the bottom tab bar, and the runner-transcript proxy route. Graphs authoring intentionally absent — it gets a "best on desktop" stub.
